# BIG Impact Methodology Brief

## The Triad: Problema · Público · Evidência
- **Problema**: a recortable, observable failure in a real flow — not a generic theme. Must name what fails (delay, abandonment, cost, repetition, escalation), where it fails, and why the status quo doesn't fix it.
- **Público**: a *specific* segment in a *concrete* context (one organisation/locale/role), never "toda a gente".
- **Evidência**: every claim carries baseline + number + source. Evidence ledger format (auditable): Sinal · O que prova · Número · Fonte · Limitação · Próximo passo.
- **Anti-patterns**: vague segment ("citizens"), opinion without source, measuring "access" instead of "resolution", solution-shaped problem statements, AI output presented as proof.

## Case Challenge Angles (pick exactly ONE recorte)
1. **Acesso sem resolução** — channel adds contacts but does not resolve; signal = low first-contact resolution, repeat contacts, escalation.
2. **Duplicação e fragmentação** — multiple doors (teleconsulta/chat, unidades móveis, linha) make the same person/case repeat across channels with no continuity.
3. **Exclusão invisível** — those who need it most don't use, don't trust, or can't access (digital inequality); high-access metrics mask under-served segments.
4. **Burnout e degradação do serviço** — program raises admin load on clinical staff, drops quality, drives turnover and more returns/escalation.

Tells: contradictory access vs. outcomes data → angle 1; same case across portas → angle 2; uptake skewed away from vulneráveis → angle 3; clinical team carga/rotatividade rising → angle 4.

## Problem Statement Checklist (mandatory)
- [ ] **Quem** — specific segment (not "everyone")
- [ ] **Onde** — concrete context (one school/município/unit/flow point)
- [ ] **O quê** — observable pain (delay, abandono, custo, repetição, escalada)
- [ ] **Frequência** — how often it occurs
- [ ] **Magnitude** — 1–2 numbers WITH source (proxies allowed if cited)
- [ ] **Status quo** — how they solve today and why it fails
- [ ] **Porquê agora** — urgency / timing
- [ ] **Hipótese causal** — likely mechanism
- [ ] **Pergunta crítica em aberto** — one open question that orients the pilot

## Evaluation Criteria (PoV Review, 29-Apr)
Scored on: relevância e clareza do problema · potencial de impacto e mensurabilidade · qualidade da solução e viabilidade · evidência/tração · equipa e execução · plausibilidade de piloto.
- Evidência > opinião
- Clareza > complexidade
- Baseline / número / fonte on EVERY claim
- Specificity > generality
- AI welcome but does NOT substitute proof
- Programa mede-se nos *assignments* (deliverables), not in discussion

## Format constraints
- Slide 1 sections: Problem Statement · Hipótese de Valor · Métricas · Tabela de Evidências
- Slide 2 sections: Pilot Sketch · Budget Range · Pressupostos
- Portuguese for slide-facing copy
- Problem Statement ≤ 500 chars; Main Hypothesis ≤ 100 chars
- Reference Problem Statement style (slide 9): "Para [segmento] em [contexto], o problema é [dor observável]: [mecanismo]. Isto [consequência mensurável] (medir [métrica X] e [métrica Y]). Hoje o status quo falha por [causa]."

## Pilot non-negotiables (from Case Challenge)
- Duration 8–12 weeks
- No budget increase, no new hires
- Privacy-first data: aggregate / anonymised preferred
- Must fit the real system (no "perfect product")
- Stakeholders with conflicting incentives: município (político), cuidados primários (operacional), hospital/urgência, comunidades vulneráveis, fornecedor tech, proteção de dados

## Required research minimums (for evidence ledger)
- 1–2 sources on induced demand / telemedicine volume effects
- 1 source on digital inequality and access to care
- 1 source on useful metrics (first-contact resolution, repetition, escalation, no-show)
- Sources cited simply (link or reference)

## Presentation format
60-second / 2-minute pitch must include: (i) read the Problem Statement, (ii) state number(s) + source(s), (iii) state the open critical question.

## Breakout roles (when collaborating)
Facilitador/a · Secretário/a · Porta-voz. Minimum entrega: Problem Statement + 1 magnitude number with source + 1 open critical question.
