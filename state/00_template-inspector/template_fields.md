# Deliverable Template Fields — Ground Truth

Generated by inspecting the actual `.pptx` and `.docx` template XML. This file is the **contract** for what fields exist. Any agent that invents a field not listed here is producing content that won't render.

When the methodology brief and this file disagree about structure, **this file wins for what fields exist**; the methodology brief still wins for *how to write inside those fields* (rigor, evidence, language).

## Evidence Notes Template (`.docx`)

The .docx is much simpler than previously assumed. Mandatory placeholders:

| Section | Field | Content |
|---|---|---|
| Header | `Notas Adicionais` (literal heading) | unchanged — section heading |
| Header | `Anexo opcional – 2 páginas no máximo` | unchanged — instruction |
| Nome do projeto/startup | `(preencher)` | "PerSense" |
| Pessoa de contacto | `(preencher)` | full name + contact |

After the headers, the body is **free-form text**, **max 2 pages**. Use it for:
- Methodology note on how evidence was curated
- Per-claim notes for EV-IDs cited on the slides (publisher, URL/DOI, internal anchor)
- Open Issues / Não Resolvido aggregated from drafts
- Chosen business thesis brief
- Alternative theses (one-line each)

The "appendix" sections we describe in `deliverable-renderer.md` all live inside this single free-form body. They are not template-named fields.

## Agent rules derived from this contract

1. `problem-definer` writes one Portuguese block ≤500 chars. The methodology brief's PS template (`Para [segmento] em [contexto]...`) is *style guidance for inside the block*, not a multi-section schema.
2. `value-hypothesis` produces one main hypothesis sentence + a single text block listing the three metrics (principal + 2 suporte). The renderer fills two text boxes, not three.
3. `evidence-table-builder` produces 4–6 lines of `Aspecto · Número · Fonte` as plain text. **Do not produce a 6-column structured table** — there is no table to fill.
4. `pilot-designer` produces three distinct assumption strings for the three Pressupostos boxes. Anything beyond three is overflow and must be moved to the Evidence Notes body.
5. `deliverable-renderer` populates `(preencher)` strings only — there are no other named placeholders in either template.

## Pilot non-negotiables (from methodology brief, reinterpreted against this contract)

The methodology brief says: *"no budget increase, no new hires"*. Reinterpretation:
- "No budget increase" = the pilot must not require the host institution (university, hospital, etc.) to increase its budget. PoV-team external funding (€3K–10K) is fine.
- "No new hires" = the pilot must run with existing PerSense team + existing host-institution staff. No commitments to expand the host's payroll.
- "Must fit the real system" = no parallel infrastructure that won't survive after the pilot ends.

State this disambiguation explicitly in `pilot.md` so reviewers don't block on ambiguity.
